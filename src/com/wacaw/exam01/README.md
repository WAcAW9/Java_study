# 📌 01 Java의 기초

## ✅ 컴파일러 언어 vs 인터프리터 언어

Java는 효율적인 개발과 실행을 위해 컴파일러와 인터프리터의 장점을 모두 취했습니다.

- **컴파일러 언어 (C, C++):** 기계어로 직접 번역하여 속도가 빠르지만, OS/CPU에 종속적이며 오류 수정 후 다시 컴파일하는 시간이 소요됩니다.
- **인터프리터 언어 (Python):** 실행 시점에 코드를 해석하여 유연하지만 상대적으로 속도가 느립니다.
    
- **Java의 특징:** * **컴파일러(`javac`)**가 소스 코드를 가상의 기계어인 **바이트코드**로 변환합니다.
    - 이 바이트코드를 **JVM(Java Virtual Machine)**이 한 줄씩 읽어 실행합니다.
        
---

## ✅ Java 실행 환경 (JDK vs JRE)

Java를 실행하고 개발하기 위한 환경 구성은 다음과 같습니다.

- **JRE (Java Runtime Environment):** JVM과 자바 프로그램 구동에 필요한 라이브러리 자원으로 구성된 **실행 전용** 환경입니다.

- **JDK (Java Development Kit):** JRE를 포함하여 컴파일러(`javac`), 디버거 등 개발에 필요한 모든 도구를 포함하는 **개발용** 세트입니다.    
    - `javac.exe`: 자바 컴파일러 (Java 코드를 바이트코드로 번역)
    - `java.exe`: JVM 실행 도구 (컴파일된 코드를 구동)

---

## ✅ Java 데이터 표현 및 변수

Java는 엄격한 타입 체크와 객체지향 구조를 가집니다.

- **구조:** Java는 **클래스(Class)** 단위로 이루어지며, 이름 중복을 방지하기 위해 **패키지(Package)**로 구분합니다.
    
- **데이터 단위:**
    - **BIT / BYTE**: 데이터 저장의 최소 단위(Bit)와 처리 기본 단위(Byte, 8 Bits).
    - **2의 보수법**: 음수를 표현하는 방식이며, 최상위 비트(**MSB**)로 부호를 결정합니다.
    - **IEEE 754**: 실수를 표현하는 부동소수점 방식으로, 가수부와 지수부로 구성됩니다.

## ✅ 타입 변환 및 연산자

### ➡️ 타입 변환 (Type Casting)

1. **자동 타입 변환**: 표현 범위가 작은 쪽에서 큰 쪽으로 갈 때 자동으로 수행됩니다.
    - `byte` → `short` → `int` → `long` → `float` → `double`
2. **수동 타입 변환**: 괄호를 사용하여 명시적으로 타입을 지정해야 합니다.
    - 예: `int value = (int) 3.14;`        

### ➡️ 연산 및 제어

- **연산자**: 사칙연산, 비교연산, 논리연산 및 단축형 연산자를 지원합니다.
- **제어문**: 프로그램의 흐름을 바꾸는 **반복문(Loop)**과 **제어문(Condition)**을 활용합니다.
